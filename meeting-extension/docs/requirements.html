Meeting Extension — Chrome拡張完結版 要件定義書

ブラウザ上の会議（Meet / Zoom Web / Teams / 任意のタブ音声）をリアルタイムに録音し、 文字起こし → 議事録生成 → タスク抽出 → Google Docs / Calendar / Sheets 書き込み → PDF / スライド書き出し → レポート / 分析画面まで、すべて Chrome 拡張機能内で完結させるツール。 既存の Mac 版 Meet パイプラインとは完全に別プロジェクトとして独立運用する。

Chrome Extension MV3 tabCapture getUserMedia Whisper API / Google STT v2 Claude Sonnet 4.6 Google Docs / Calendar / Sheets PDF / Slides Export
1

背景・目的

このプロジェクトのモチベーション

既存の Mac 版 Meet パイプラインは「Drive に上がった録画ファイルを後処理する」前提だった。 本プロジェクトはそれとは別に、ブラウザを「ブラックホール」として使い、 会議中にタブ音声とマイク音声を その場で両方拾い、リアルタイムに処理する 拡張機能を作る。 Meet に限定されず、ブラウザで音が鳴る会議ツールならどれでも対応できる点が最大の差別化。

新システム
Chrome 拡張機能（MV3）単体で録音〜出力〜分析まで完結
対象会議ツール
Google Meet / Zoom Web / Microsoft Teams Web / Discord Web / その他「ブラウザで音が鳴る」もの全般
既存 Mac 版との関係
完全に切り離し。別プロジェクト・別ディレクトリ・別 OAuth クライアント・別台帳。データ共有なし。
非目標（v1 スコープ外）
商用配布 / Chrome Web Store 公開 / 複数ユーザー対応 / Workspace（法人）対応 / モバイル対応 / 真の話者分離（diarization）/ 録音ファイルのクラウド長期保存 / 120分超の自動分割 / 会議タブの URL 自動検出（v2 以降）
前提
会議中ずっと Mac が起動し、Chrome が開いており、対象タブが閉じられないこと。機密会議は対象外（運用ルールで除外）。
2

ワークフロー全体図

STEP 1
録音開始 Popup の「このタブを録音」ボタンを押す。tabCapture（タブ音声）+ getUserMedia（マイク）の 2 トラック並列録音を開始。タイトル入力ダイアログを表示。
STEP 2
チャンク化 60 秒ごとに MediaRecorder のチャンクを切り出し、IndexedDB に逐次保存。
STEP 3
逐次文字起こし チャンク単位で文字起こし API に送信。source_label（self / tab）付きで segments を蓄積。
STEP 4
録音停止 会議終了 or 手動停止で全 segments を結合。フル原文を Side Panel に表示。
STEP 5
構造化 Claude Sonnet 4.6 で議事録 / タスク / 改善点 / 決定事項を JSON 化。Side Panel にプレビュー。
STEP 6
承認 手動モード = 1 件ずつ承認 / 自動モード = 全件 approved にして次へ。設定で切り替え。
STEP 7
Google 系書き込み Docs（原文 + 議事録 2 本） / Calendar（承認タスク） / Sheets（台帳 1 行）を拡張機能から直叩き。
STEP 8
書き出し 議事録を PDF / スライド（pptx）に書き出し（任意）。ローカル DL or Drive 保存。
STEP 9
レポート / 分析 過去ミーティング一覧 / 累積時間 / タスク完了率 / 頻出キーワード等を Report ページで可視化。

データフロー（送信先の定義）

外部に出るデータと送信先を明示する

v1 は個人前提でこの送信先を受容する。機密会議は本拡張の対象外とする。

データ種別 送信先 用途 保管期間
音声バイナリ（webm/wav） 文字起こし API（OpenAI Whisper または Google STT v2） 文字起こし API 側ポリシーに従う
文字起こし本文（テキスト） Anthropic Claude API / Google Docs（原文 Doc） 構造化 / 保管 Doc は無期限（ユーザー Drive 内）
議事録 JSON Google Docs / Google Sheets / Google Drive（PDF/pptx 保存時） 保管 / 共有用 無期限（ユーザー Drive 内）
タスクメタデータ Google Calendar / Google Sheets スケジュール / 台帳 無期限
ミーティング状態 / 履歴 / 設定 / API キー chrome.storage.local（端末ローカル） 状態管理 / 設定 無期限（ユーザー削除可）

機能要件（FR）

MUST

FR-01 タブ音声 + マイクの並列録音

会議中のタブ音声とマイクを別トラックで同時録音。トラック由来ラベル（self=マイク / tab=タブ音声）を付与する。真の話者分離はしない。

  • API: chrome.tabCapture / getUserMedia
  • 場所: Offscreen Document
  • 形式: WebM/Opus → 必要に応じて wav 変換
MUST

FR-02 任意の会議ツール対応

Meet / Zoom Web / Teams / Discord 等、特定ツールに限定せず「現在のタブ音声」を拾える。

  • v1 は手動で「このタブを録音」ボタンのみ。URL 自動検出は v2 以降
  • tabCapture はユーザー操作起点でのみ開始できる MV3 制約を受容
  • タブの title / url 取得は activeTab 権限で録音開始時に 1 回取得
MUST

FR-03 リアルタイム逐次文字起こし

録音チャンク（60 秒）ごとに文字起こし API へ送信し、Side Panel に追記表示。

  • v1 では「OpenAI Whisper API または Google STT v2 のいずれか 1 つを実装すること」を満たせばよい（Phase 1 PoC で評価して決定）
  • source_label: self / tab
  • タイムスタンプ付き segment
  • 日本語必須
MUST

FR-04 議事録 / タスクの構造化

録音終了後、Claude Sonnet 4.6 で要約・タスク抽出・改善点・決定事項を JSON で取得する。

  • LLM: Anthropic Claude Sonnet 4.6（確定）
  • 個人前提、送信前承認ゲートなし
MUST

FR-05 Google Docs 出力

文字起こし原文 Doc と議事録 Doc の 2 本を指定フォルダに作成。

  • 命名: [文字起こし] YYYY-MM-DD タイトル
  • タイトル取得: 開始時に tab_title を初期値として表示し、ユーザーが編集可能。空の場合は 無題ミーティング YYYY-MM-DD HH:MM にフォールバック
  • appProperties で meeting_uuid をタグ付け
MUST

FR-06 承認モード（手動 / 自動）

抽出タスクの承認方式を設定で切り替え可能にする。

  • 手動承認モード（デフォルト）: 1 件ずつ y / n / 編集
  • 自動承認モード: 全件 approved として次工程へ即進む
  • 編集可能項目: タイトル / 日時 / 所要時間 / メモ
  • 承認単位: 1 ミーティング
MUST

FR-07 Google Calendar 登録

承認タスクのみ Calendar に登録。再実行で重複しない。

  • client-supplied event ID（task_uuid → base32hex）
  • 409 Conflict は成功扱い
MUST

FR-08 Sheets 台帳追記

1 ミーティング 1 行で本拡張専用の Sheet に追記（Mac版と非共有）。

  • meeting_uuid ユニーク制約（read-before-write）
  • 列: 日付 / タイトル / 録音時間 / ツール / Docs URL / タスク数 / status
MUST

FR-09 レポート / 分析画面

Report ページで過去ミーティングを一覧 / 集計 / 検索できる。

  • サマリーカード / グラフ / 一覧 / 分析 / フィルタ / エクスポート（詳細は 9 章）
MUST

FR-10 議事録の書き出し

議事録を PDF と pptx で書き出せる。

  • PDF: jsPDF（拡張内同梱・確定）
  • Slides: PptxGenJS（拡張内同梱・v1 確定。Google Slides API は v2 で再検討）
  • 保存先: ローカル DL / Drive のいずれか選択
MUST

FR-11 タスク完了状態の更新

Calendar 登録だけでは完了状態にならないため、Report 画面でタスクを done に手動更新できる。

  • done への遷移は「Report 画面のチェックボックス」で実行
  • completed_at を記録し、タスク完了率の集計源とする
v2 FUTURE

FR-12 会議タブの URL 自動検出

v1 では実装しない。tabs 権限追加とバックグラウンド常駐が必要なため、v2 以降で検討。v1 は手動ボタンのみ。

MUST

FR-13 失敗チャンクの再送

文字起こし API が失敗したチャンクを Report 画面の「失敗一覧」からまとめて再送できる。（18章の受け入れ基準に含まれるため MUST に格上げ）

  • 失敗チャンクは IndexedDB の永続キューに保存
  • Report 画面から 1 クリックで再送
SHOULD

FR-14 ブラウザ通知

文字起こし完了 / 承認待ち / 失敗を chrome.notifications で通知。

MAY

FR-15 議事録の手動編集

Side Panel 上で議事録の「タイトル / 本文段落 / 箇条書き項目」のテキスト編集ができる（書式は不変）。

MAY

FR-16 録音音声のローカル DL

後から聞き直せるよう、生の音声ファイルを IndexedDB から DL できる。

状態遷移（FSM）

1 ミーティングは下記 8 状態のいずれかに必ず属する

失敗状態 *_failed を各正常状態の対になる形で持つ。再試行は同じ正常状態に戻ることで実現。

recording
   ↓ 録音停止
transcribing  ──(失敗)──> transcribing_failed ──(retry)──> transcribing
   ↓ 全チャンク文字起こし完了
structuring   ──(失敗)──> structuring_failed  ──(retry)──> structuring
   ↓ Claude 応答受領
awaiting_approval   ←─(自動承認モードは即通過)
   ↓ 承認完了
writing_docs        ──(失敗)──> writing_docs_failed     ──(retry)──> writing_docs
   ↓
writing_calendar    ──(失敗)──> writing_calendar_failed ──(retry)──> writing_calendar
   ↓
writing_sheets      ──(失敗)──> writing_sheets_failed   ──(retry)──> writing_sheets
   ↓
done

再試行・冪等の原則

  • writing_docs：Doc の appProperties で meeting_uuid 検索 → 既存があれば再利用
  • writing_calendar：task_uuid 由来の決定論 event ID で 409 = 成功扱い
  • writing_sheets：read-before-write で meeting_uuid 行を検索
  • いずれの *_failed からも、ユーザー操作 or 自動リトライで同じ正常状態に戻る
  • done に到達するのは writing_sheets まで完走したときのみ

排他制御

  • recording 中は新規録音を開始できない（chrome.storage.local の lock キー）
  • 同一 meeting_uuid に対する API 書き込みは Service Worker 内のキューで直列化
  • lock 取得失敗は例外で即終了（中途半端な yield を作らない）
6

重複防止の粒度

レベルキー防げるケース防げないケース
L1: セッション重複 meeting_uuid 同一録音の再処理 / 書き込み失敗からのリトライ 同じ会議をうっかり 2 回録音した場合
L2: 実会議重複 tab_url + started_at の ±15 分窓 + title 類似度 ≥0.8 同じ会議を 2 回録音した場合の警告（Side Panel に「類似ミーティングがあります」と表示し、新規 / マージ / 上書きを選ばせる） 完全別タブで意図的に再録音した場合（このときは別ミーティング扱い）

非機能要件（NFR）

区分要件指標 / 閾値（測定条件）
性能（文字起こし） 60 秒チャンクの文字起こし応答が録音追従できる チャンク 60 秒 / 16kHz mono wav / OpenAI Whisper API / 同時送信 1 本 / 上り 20Mbps 以上 → 応答 ≤ 15 秒
SLA（手動承認モード） 承認完了後の Docs/Calendar/Sheets 書き込みが完了するまで 承認操作完了から 5 分以内
SLA（自動承認モード） 録音停止から Docs/Calendar/Sheets 書き込み完了まで 録音停止から 10 分以内（60 分会議の場合）
可用性 会議中ずっと Chrome が起動している前提 Mac OFF / Chrome 閉じ = 動かない（前提として受容）
冪等性 同じ meeting_uuid で 2 回処理しても Docs / Calendar / Sheets が重複しない FSM + 決定論的 event ID + read-before-write
クラッシュ耐性 録音中に Service Worker が死んでも Offscreen Document で録音継続 chunk 単位で IndexedDB に逐次永続化、state は chrome.storage.local
セキュリティ 3 章のデータフロー表に定義した送信先以外には送らない OAuth は chrome.identity / API キーは端末ローカル平文保存（受容）
コスト 1 ミーティング 60 分あたり $0.5 以下 Whisper API（$0.006/分 × 60 = $0.36）+ Claude Sonnet 4.6 入力 60K トークン / 出力 4K トークン 想定 → 合計 ≤ $0.5
容量上限 1 ミーティング 120 分まで 超過時は自動分割（60 分 × N）して別 meeting_uuid として保存
ストレージ chrome.storage.local は 10MB 以内に収める メタデータのみ。音声と segments は IndexedDB（端末ストレージ上限の範囲）

アーキテクチャ

方針

Service Worker は短命だが永続的に存在する性質、Offscreen Document は長時間動かせるが UI を持たない性質を組み合わせる。 録音実体は Offscreen、API 呼び出しは Service Worker、UI は Popup と Side Panel と Report ページが担当。 音声バイナリと segments は IndexedDB、状態とメタデータは chrome.storage.local に分離保存する。 Cloud Run / Apps Script は一切使わない。

データ・状態モデル

永続化は chrome.storage.local + IndexedDB + Sheets 台帳の三層

chrome.storage.local は state machine とメタデータのみ（10MB に収める）。 音声 / segments / 議事録 JSON は IndexedDB に逃がす。Sheets 台帳は中長期の履歴と分析の源泉。

meetings（chrome.storage.local メタのみ）

  • meeting_uuid（PK）
  • state（5 章 FSM）
  • started_at / ended_at
  • tab_url / tab_title（録音元）
  • tool（meet / zoom / teams / discord / other）
  • title（編集後の最終タイトル）
  • transcript_doc_id / minutes_doc_id
  • ledger_row_index
  • error / updated_at

recordings（IndexedDB）

  • meeting_uuid（FK）
  • chunk_index / source_label（self/tab） / blob
  • transcript_segments[]（タイムスタンプ + テキスト）
  • minutes_json（Claude 出力）

tasks（chrome.storage.local + Sheets 台帳）

  • task_uuid（PK）
  • meeting_uuid（FK）
  • title / owner（自由文字列） / due / estimate_minutes / context
  • status（pending / approved / skipped / inserted / done）
  • calendar_event_id
  • approved_at / inserted_at / completed_at

Sheets 台帳（meetings_log / tasks_log 2 シート）

  • meetings_log: meeting_uuid / 日付 / タイトル / 時間 / ツール / Docs URL / タスク数 / 完了数 / status
  • tasks_log: task_uuid / meeting_uuid / title / owner / due / status / event_id / completed_at
  • read-before-write で重複防止

異常系ユースケース

ケース 挙動 / 状態遷移 ユーザー通知
マイク権限拒否 タブ音声のみで録音継続（source_label は tab のみ） Popup に「マイクなしで録音中」バッジ

採用しない選択肢とその理由

選択肢不採用理由
Apps Script / Cloud Run を併用 本プロジェクトは「拡張機能完結」がコンセプト。Mac 版とは設計思想が違うため混ぜない。
Service Worker だけで録音 SW は数十秒〜数分でアイドル停止する。長時間録音は Offscreen Document に逃がす。
Drive 録画を待つ 本プロジェクトはリアルタイム派。Drive 録画ファイル経路は Mac 版に任せる。
ローカル Whisper（WASM） 120 分音声を WASM で回すとメモリ的に厳しい。v1 は API に逃がす。v2 で再検討。
真の話者分離（diarization） v1 は source_label（self/tab）のみ。複数相手は tab に潰す。v2 以降で再検討。
Chrome Web Store 公開 個人 unpacked 配布で十分。審査コストを払わない。
Mac 版と台帳共有 記録単位（録画ファイル vs 録音セッション）が違うため、混ぜると分析が破綻する。別シート運用。

リスクと前提（受容するもの）

  • Chrome / Mac が起動していないと録音できない：これは本プロジェクトの前提として受容。
  • Service Worker のアイドル停止：Offscreen Document に録音実体を逃がすことで対策。
  • tabCapture はタブを閉じると止まる：会議タブは閉じない運用ルールで受容。閉じた場合の挙動は 12 章。
  • マイクの権限ダイアログ：初回録音時に必要。拒否時はタブのみで継続。
  • 音声 / 文字起こしは外部 API へ送られる：3 章のデータフロー表のとおり。機密会議は対象外。
  • API コスト：60 分 1 本で約 $0.3〜$0.5 を見込み。月 30 本で $9〜$15。
  • API キーは端末ローカル平文保存：個人運用のリスク受容として明記。
  • Chrome 更新で MV3 仕様が変わるリスク：Offscreen / sidePanel は比較的新しい API。定期的に動作確認。

受け入れ基準（v1 完了の定義）

  • Meet / Zoom Web / Teams のいずれかで開始した会議で、タブ音声とマイクの両方が source_label 付きで文字起こしされる。
  • 手動承認モードでは「承認完了から 5 分以内」、自動承認モードでは「録音停止から 10 分以内」に Docs 2 本 / Calendar タスク / Sheets 台帳行が揃う。
  • 同じ meeting_uuid を 2 回処理しても、Docs / Calendar event / 台帳行が重複しない。同じ会議をうっかり 2 回録音した場合は writing 前に L2 重複検出ダイアログが出る。
  • 議事録を PDF と pptx で書き出せる（ローカル DL と Drive 保存の両方）。
  • Report 画面で過去 30 日のミーティングが一覧 / 集計 / フィルタでき、タスク完了チェックで done に遷移できる。
  • 12 章の異常系ユースケース 13 件のうち、MUST 対応 11 件（無音検知と類似検出を除く全件）で定義どおりの挙動を示す。
  • 失敗チャンクが Report 画面に表示され、ボタン 1 つで再送できる。
  • 11 章の必須設定が未充足の場合、録音ボタンが disabled になり Settings へ誘導される。
  • Codex 敵対的レビューを最低 2 ラウンド実施し、high 以上の指摘がゼロになる。
