your voice, your Mac.

04 / FLOW

録音から議事録まで、
ぜんぶ自動で連なる。

ボタンひとつで起動するパイプライン。失敗してもリトライキューが自動で拾います。

how it works

① 会議タブを選んで「録音開始」

ツールバーの拡張アイコン → タイトルを入れて開始。タブ音声＋マイクが offscreen ドキュメントで同時録音されます。最大 120 分で自動停止。

1
2

② 60秒チャンクで faster-whisper に送る

音声は IndexedDB にも保管されつつ、localhost:9000 に POST。Mac 内で逐次文字起こしが進み、Side Panel にリアルタイム表示されます。

③ 停止 → Gemini が議事録 JSON を作る

録音を停止すると、文字起こし全文を Gemini に投げて要約 / 決定事項 / 改善点 / タスクを抽出。Side Panel にプレビューが出ます。

3
4

④ タスクを承認

Side Panel でタスクを編集／承認／スキップ。自動承認モードを選べばこのステップを飛ばせます。

⑤ Docs / Calendar / Sheets に書き込み

文字起こし Doc・議事録 Doc・Calendar イベント（task_uuid → event_id でべき等）・Sheets 台帳の 4 出力。失敗しても chrome.alarms 永続キューで自動リトライ。

06 / FAQ

よくある質問

good to know
本当に月額0円で運用できますか？

はい。文字起こしは Mac 内の faster-whisper、議事録生成は Gemini の無料枠（1日1500req）、書き出しは Google の個人無料枠で完結します。重い会議を月20本走らせても無料枠で足ります。

Intel Mac でも動きますか？

動きます。実機検証は MacBook Pro 13" 2020 / Core i7 / 16GB で行っています。文字起こしは "small" モデル + int8 量子化で 60秒音声を 30〜45 秒程度で処理。Apple Silicon ならもっと速くなります。

Google Meet 以外でも使えますか？

使えます。Zoom Web / Microsoft Teams / Discord / YouTube など、Chrome のタブで音が鳴るならなんでも対象です。タブ音声＋マイクをそれぞれ別ラベルで保存するので、自分の発言と相手の発言を切り分けられます。

会議の内容が外部に送られませんか？

音声と文字起こしテキストは Mac 内で完結します。Gemini で議事録を生成するときだけ、文字起こしのテキストが Google に送られます（API 規約上、学習に使われる可能性あり）。完全に外部送信ゼロにしたい場合は Apple Silicon Mac + Ollama 構成への切り替えが可能です。

120分を超える会議はどうなりますか？

120分で自動停止し、通知が出ます。新規録音セッションとして手動で再開してください（自動分割はせず、1会議 = 1議事録の単位を維持する仕様です）。

失敗したときの再実行は？

chrome.alarms + IndexedDB で永続的なリトライキューを持っています。文字起こし失敗 / 議事録生成失敗 / Google 書き込み失敗のいずれも 1分周期で指数バックオフ（最大30分）で自動リトライ。Report 画面から手動で再実行もできます。

Mac の他のアプリの音は録れますか？

いいえ、対象は Chrome タブの中で鳴っている音だけです。デスクトップアプリ版 Zoom や Teams など、ブラウザ外のアプリの音はキャプチャされません（Chrome 拡張の仕組み上の制約）。
