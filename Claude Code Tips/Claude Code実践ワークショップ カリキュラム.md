# Claude Code実践ワークショップ カリキュラム

---

## 第1部：Skillsで"一度やった仕事"を資産化する（40分）

### (a) Skillsとは何か — 業務テンプレートの概念
- 「毎回同じ指示を出す」のをやめる方法
- Skillsの構造（SKILL.md）と作り方
- 実例：議事録→提案書を一発生成するSkill

### (b) ワーク①：自分専用のSkillを作る
- 自分の繰り返し業務を1つ選び、Skill化する
- テンプレート設計 → テスト → 改善のサイクルを体験
- 「次から"これやって"の一言で終わる」状態を作る

---

## 第2部：MCP連携で外部ツールと繋ぐ（35分）

### (a) MCPとは — AIと外部サービスの接続規格
- Google（Gmail・Calendar・Drive・Sheets）との連携
- Notion連携 — タスク管理・DB操作をClaude Codeから実行
- Slack連携 — メッセージの送受信・チャンネル監視

### (b) デモ：Claude Codeから外部ツールを操作
- 「来週の予定を教えて」→ Googleカレンダーを参照して回答
- 「この内容でSlackに投稿して」→ 指定チャンネルに送信
- 「Notionのタスク一覧を更新して」→ DBを直接操作

## 第3部：Agent Teams & OpenClaw — 複数AIの並列稼働（45分）

### (a) Agent Teams — 1つの指示で複数AIを同時に動かす
- Agent Teamsの仕組みと使いどころ
- 実例：市場調査レポートを3つのAIが並列で作成
- デモ：「この3社を調べて比較表を作って」を並列実行

### (b) ワーク②：Agent Teamsで調査タスクを実行
- お題を決めて、複数エージェントに分担させる
- 結果を統合してレポート化するまでを体験

### (c) OpenClaw — Claude Codeを24時間稼働させる
- OpenClawとは：VPS上で動くClaude Code常駐エージェント
- 複数エージェントの役割分担（メイン応答・ログ記録・監視）
- Slack・LINE等のメッセージに自動応答するBotの構築例
- 「自分が寝ている間もAIが仕事をしている」状態の作り方

---

## 第4部：自動化とアプリ開発（40分）

### (a) Cronジョブ — 定型業務の完全自動化
- 「毎朝○時にこれを実行」をClaude Codeで設定
- 日報生成・データ収集・リマインドの自動化パターン
- 実例：毎朝のニュース収集 → SNS投稿ドラフト自動生成

### (b) ワーク③：Webアプリを作ってデプロイする
- 「こんなツール欲しい」をClaude Codeに伝える
- Webアプリを生成 → ローカルで動作確認
- Vercelにデプロイして公開するまでを一気通貫で体験
- プログラミング経験不要 — AIが全部書いてくれる

## 第5部：運用と次のステップ（10分）
- Memory / Context管理 — AIを長期的に「育てる」方法
- コスト最適化 — Pro・Max・APIの使い分け
- 「AI駆動の仕事術」ロードマップ

## こんな方におすすめ
- Claude Codeの基本操作は分かるが、活用の幅が広がっていない
- 繰り返し作業をSkills化して自動化したい
- Google・Notion・Slack等の外部ツールとAIを連携させたい
- 複数のAIを同時に動かして業務を並列処理したい
- Claude Codeを常駐させて24時間動く仕組みを作りたい
- 自分専用のWebアプリやツールを作って公開してみたい
- AI時代の「仕組みを作れる側」になりたい
