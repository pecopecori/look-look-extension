## Claude Codeとは？

ターミナルで実行される「agentic（自律的）アシスタント」。  
コードを読み、ファイルを編集し、コマンドを実行し、ウェブを検索する。

### agentic ループ（3フェーズ）
1. **コンテキストの収集** — 必要な情報を集める
2. **アクションの実行** — 作業を行う
3. **結果の検証** — 正しいか確認する

この3フェーズをループしながら仕事を進める。

## 11の重要機能

### ① CLAUDE.md — AIへの「設定書」

プロジェクトフォルダに置くだけで、セッション開始時に自動で読み込まれる設定ファイル。

**使用例：**
- 「常にテストも一緒に作って」
- 「コメントは日本語で書いて」
- 「このフォルダは絶対に触るな」

一度書けば毎回説明不要。`/init` で自動生成も可能。

> 🔗 [指示とメモリを保存する（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/memory)

---

### ② Permissions — 安全に動かす

AIが動くたびに「本当にいいですか？」と確認する権限システム。

**カスタマイズ例：**
- テストの実行 → 毎回確認しなくていい（許可リストに追加）
- ファイルの削除 → 絶対に許可しない（ブロックリストに追加）

速度と安全性のバランスを自分で決められる。最初は厳しめに設定して、信頼が積み上がったら開放していくのが推奨。

> 🔗 [パーミッションモード（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/security)

---

### ③ Plan Mode — 考えてから動く

`Shift + Tab` で切り替え。「読む・考える・提案する」だけで、変更は一切加えないモード。

**フロー：**
1. ファイルを読み込む
2. 質問してくる
3. 段階的な計画を提案する
4. 確認したら通常モードに戻して実行

料理で言うと、レシピを確認してから調理を始める感じ。大きな作業の前に必ず使いたい機能。

> 🔗 [Plan Modeの使い方（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/plan-mode)

---

### ④ Checkpoints — 失敗を恐れずに実験できる

Claudeが編集するたびに自動でスナップショットを撮る機能。

**使い方：**
- `/re` → タイムスタンプ付きのチェックポイント一覧を表示
- 戻りたい時点を選ぶだけ

「失敗してもいつでも戻れる」という安心感で、試せる範囲が一気に広がる。

> 🔗 [チェックポイントの仕組み（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/checkpoints)

---

### ⑤ Skills — 繰り返しをゼロにする

`skill.md` に手順を一度書いておくと、「このスキルを使って」と言うだけで動く再利用可能なプロンプト。

**メリット：**
- チームで共有できる
- 長いプロンプトを暗記しなくていい
- 一度作れば何度でも使える

AIへの指示書を資産として積み上げていく感覚。

> 🔗 [スキル（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/skills)

---

### ⑥ Hooks — 自動で動くスクリプト

AIが動くたびに自動で走るスクリプトを設定できる。

**設定例：**
- コードを書き換えるたびにフォーマット（コード整形）を自動でかける
- セキュリティチェックを自動で走らせる
- ログを自動で記録する

「ツールを呼び出す前」「呼び出した後」などタイミングも指定可能。  
「絶対に忘れてはいけない確認作業」をAIの動作に組み込んでしまう発想。

> 🔗 [フック（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/hooks)

---

### ⑦ MCP（Model Context Protocol）— 外部ツールとつなぐ橋

Claude CodeをFigmaやSlackなど外部ツールに接続するオープンプロトコル。  
MCPサーバーを追加すると、そのサーバーが持つすべてのツールにアクセス可能になる。

公開されているMCPサーバーはすでに数千個。エンジニアじゃなくても、選んで追加するだけで使える。

> 🔗 [MCP（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/mcp)

---

### ⑧ Plugins — 設定をまとめて配布

スキル、Hooks、MCPサーバー、サブエージェントの設定をまとめて「1コマンドでインストール」できる形にしたもの。

**特徴：**
- マーケットプレイスで公開できる
- GitリポジトリのURLを渡すだけでインストールできる

個人の試行錯誤を、組織の資産に変える仕組み。

> 🔗 [プラグイン（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/plugins)

---

### ⑨ Context Management — AIの記憶を管理する

Claude Codeは最大約20万トークンのコンテキストウィンドウで動く。会話が長くなると満杯になる。

**コマンド：**
- `/context` → 現在の使用量を確認（CLAUDE.md・スキル・MCP設定・会話履歴の内訳も確認可能）
- `/compact` → 手動で圧縮

満杯に近づくと自動で圧縮してくれる。

> 🔗 [コンテキストウィンドウ（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/context)

---

### ⑩ Slash Commands — ショートカット集

`/`（スラッシュ）+コマンド名で即実行できるキーボードショートカット集。

| コマンド | 説明 |
|---|---|
| `/init` | CLAUDE.mdを自動生成 |
| `/re` | チェックポイント一覧を表示 |
| `/context` | コンテキスト使用量を確認 |
| `/compact` | 会話を手動で圧縮 |
| `/permissions` | 権限設定メニューを開く |
| `/agents` | サブエージェントを作成 |

`/` と入力すると利用可能なすべてのコマンドが表示される。

> 🔗 [組み込みコマンド（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/cli-reference)

---

### ⑪ Subagents — 複雑な仕事を「チーム」に任せる

特定のジョブに特化した独立したClaudeセッション。`/agents` コマンドで作成。

**フロー：**
1. メインセッションから仕事を渡す
2. サブエージェントが独立して作業する
3. 短い要約だけメインに返ってくる

メインの会話がクリーンに保たれる。一つのセッションで全部やろうとすると会話が混乱してコンテキストも圧迫されるのを防ぐ。

> 🔗 [サブエージェント（公式ドキュメント）](https://docs.anthropic.com/ja/docs/claude-code/sub-agents)
