🚀 はじめての方へ｜Google連携セットアップ手順

Sheets連携（スプシ同期）と Calendar連携（予定の自動投入・空き時間検索）の両方で、この手順が最初の一度だけ必要です🌷
所要時間：10〜15分くらい。途中で迷ったら各ステップの「💡困ったら」を見てください。
※ 既に Sheets で動いている方は、STEP 2-2 だけ追加すれば Calendar も使えます。

🛠 よく使うショートカット

↑「拡張機能フォルダを確認」を押すと chrome://extensions/ が開きます。
そこで「読み込み元」のパスをクリックすると、Finderで拡張機能のフォルダが開きます🌷

1
Google Cloud Console でプロジェクトを作る → 開く

📌 これは何？

Google Cloud Console（クラウドコンソール）は、Googleが提供するAPI管理画面のこと。
「プロジェクト」というのは、API利用権のひとまとまりのようなものです。 （例：TimeCardプロジェクト＝TimeCardが使うAPIをまとめておく箱）

🎯 なぜ必要？

これから「Sheets API」「Calendar API」を使えるようにするのですが、その受け皿になる場所が必要です。プロジェクトを作ってあげると、その中でAPIを有効化したり、鍵を発行したりできます。

✏️ 手順

① 上の「→ 開く」リンクから Google Cloud Console を開く
② Googleアカウントでログインする（はじめての方は利用規約に同意）
③ 画面上部の「プロジェクトを選択」というプルダウンをクリック
④ 出てきたダイアログの右上「新しいプロジェクト」をクリック
⑤ プロジェクト名を入力（例：TimeCard、英数字でも日本語でもOK）
⑥ 「作成」ボタンを押す
⑦ 作成完了の通知が出たら、もう一度上部プルダウンを開いて、今作ったプロジェクト名を選択した状態にする
⑧ 画面左上に「TimeCard」と表示されたらOK

💡 困ったら

・「組織を選択」と出た→「組織なし」でOK
・有料登録を求められない？→無料枠で十分動きます。クレジットカード登録も不要
・プロジェクト名は後から変えられる？→変えられます。最初はテキトーでも大丈夫

2
使うAPIを2つ有効化する

📌 これは何？

「API」は、Googleのサービス（SheetsやCalendar）に対して「データを書き込んで」「予定を取って」と依頼する窓口です。
デフォルトでは閉まっているので、「使うよ」と宣言する作業＝有効化が必要です。

🎯 なぜ必要？

・Sheets APIを有効化しないと → 稼働データのスプシ書き込みができない
・Calendar APIを有効化しないと → カレンダーへの予定投入・空き時間検索ができない
どっちか片方だけでも動きますが、両方使うなら両方有効化が必要です。

✏️ 手順（2-1: Sheets API）

① 下のリンクをクリックで開く
　→ Sheets API を開く
② 画面左上に STEP 1 で作ったプロジェクト名が表示されていることを確認
　（違うプロジェクトが表示されていたらプルダウンで切り替え）
③ 「Google Sheets API」のタイトル下にある青い「有効にする」ボタンをクリック
④ 「APIが有効化されました」と表示されたらOK

✏️ 手順（2-2: Calendar API）

① 下のリンクをクリックで開く
　→ Calendar API を開く
② 同じく左上のプロジェクト名を確認
③ 「Google Calendar API」の青い「有効にする」ボタンをクリック
④ 有効化完了のメッセージが出たらOK

💡 困ったら

・「有効にする」ボタンが見つからない→もうすでに有効化済みかも（「管理」ボタンに変わってる）
・プロジェクトが違う→左上のプロジェクト名プルダウンでSTEP1のプロジェクトに切り替え
・API名が違う→「Google Sheets API」「Google Calendar API」と書いてあるか念のため確認

3
この拡張機能の「アイテムID」をコピーする

📌 これは何？

アイテムID＝Chrome拡張機能ごとに割り振られた32文字の英字のことです。
例：abcdefghijklmnopqrstuvwxyzabcdef
（マイナンバーや住所みたいに、その拡張機能を一意に識別するためのIDです）

🎯 なぜ必要？

次のSTEP 4で「鍵（OAuthクライアントID）」を作るのですが、
Googleに「この鍵はどの拡張機能用ですか？」と聞かれます。そのときアイテムIDで紐付けしないと、認証時にエラーになります。

✏️ 手順

① Chromeのアドレスバーに chrome://extensions/ をコピペして開く
② 開いたページの右上にある「デベロッパーモード」というトグルをONにする
　（これがOFFだとIDが表示されません）
③ 「TimeCard Pro」のカードを探す
④ カードの中に「ID: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx」と書かれている32文字をマウスで選択
⑤ Cmd+C（Mac）/ Ctrl+C（Win） でコピー
⑥ メモ帳・Sticky Notes・iPhoneのメモなどに一旦貼り付けておく（STEP 4で使うので）

💡 困ったら

・IDが見つからない→「デベロッパーモード」がONになっているか再確認
・「TimeCard Pro」が見つからない→拡張機能を有効化していますか？（カード左下のトグル）
・IDが32文字より短い→コピーミス、もう一度全選択
・IDが変わってしまった！→開発中の拡張は再読み込みでIDが変わることがあります。その場合はSTEP 4 でクライアントIDを作り直すか、ID欄を更新

4
OAuthクライアントID（鍵）を発行する → 開く

📌 これは何？

OAuthクライアントID＝「TimeCardがあなたのGoogleアカウントにアクセスしていいよ」という許可証のようなものです。
形式：123456789-abcdef.apps.googleusercontent.com
（長いですが、ただの文字列なのでコピペすればOK）

🎯 なぜ必要？

この鍵を manifest.json に書いておくことで、TimeCardがあなたのSheetsやCalendarにアクセスできるようになります。
鍵がないと、Googleは「あなた誰？」となってアクセスを拒否します。

✏️ 手順

① 上の「→ 開く」リンクから「APIとサービス」→「認証情報」を開く
② 左上のプロジェクト名がSTEP 1のものか確認
③ 画面上部の「+ 認証情報を作成」ボタンをクリック
④ 出てきたメニューから「OAuth クライアント ID」を選択
⑤ ★ もし「同意画面を構成」と出たら（はじめての方）→ 下の「同意画面の作り方」を先にやってから戻ってください
⑥ 「アプリケーションの種類」プルダウンから「Chrome 拡張機能」を選択
⑦ 名前：何でもOK（例：TimeCard クライアント 1）
⑧ 「アイテムID」欄に、STEP 3 でコピーした32文字を貼り付け
⑨ 一番下の「作成」ボタンをクリック
⑩ 「OAuth クライアントを作成しました」というダイアログが出る
⑪ 「クライアント ID」と表示されている長い文字列（〇〇〇.apps.googleusercontent.com）の右のコピーアイコンをクリック
⑫ メモ帳などに貼り付けておく（STEP 5で使う）

🔧 同意画面の作り方（はじめての方のみ）

「同意画面を構成」ボタンが出たら、次の流れでサクッと作れます：
① User Type: 外部を選択 → 「作成」
② アプリ名：何でもOK（例：TimeCard）
③ ユーザーサポートメール：あなたのGmailアドレスを選択
④ 開発者の連絡先：同じくあなたのGmailアドレス
⑤ 「保存して次へ」を押してスコープ画面へ（スコープ追加はスキップでOK）
⑥ テストユーザー画面で必ず「+ ADD USERS」をクリックし、あなた自身のGmailアドレスを追加（重要！）
　→ これを忘れると STEP 6 で「アクセスをブロック」エラーが出ます
⑦ 「保存して次へ」を押して完了 → STEP 4の④に戻る

⚠️ よく出るエラー：「アクセスをブロック」

認可時に「TimeCard は Google の審査プロセスを完了していません」と出る場合は、テストユーザー登録が抜けています。
① → OAuth同意画面を開く
② 「テストユーザー」セクションまでスクロール
③ 「+ ADD USERS」または「+ ユーザーを追加」をクリック
④ あなたのGmailアドレスを入力 → 保存
⑤ TimeCardでもう一度認可を試す

💡 困ったら

・「アイテムIDが無効です」と出る→32文字ちょうど？余計な空白が入っていないか確認
・「アクセスがブロックされました」と出る→同意画面の作成がまだ。「同意画面の作り方」をやり直し
・クライアントIDのダイアログを閉じてしまった→認証情報一覧に戻り、作ったクライアントIDを再度開ける
・テストユーザーの追加が必要？→自分のGmailアドレスをテストユーザーに登録しておくと安心

5
manifest.json にクライアントIDを貼り付ける

📌 これは何？

manifest.json は、Chrome拡張機能の設計書のようなファイル。
「何のAPIを使うか」「どんな権限が必要か」「どの鍵を使うか」が書いてあります。
ここにあなた専用のクライアントIDを書き込みます。

🎯 なぜ必要？

manifest.json の oauth2.client_id に、STEP 4 で作ったあなた専用のIDが入っている必要があります。
もし "YOUR_CLIENT_ID..." のような仮の値になっていたら、認証は動かないのでここで置き換えます。

✏️ 手順（フォルダの場所が分からない人もこれで大丈夫）

《拡張機能フォルダを開く》
① 上の「📁 拡張機能フォルダを確認」ボタンをクリック
　（または Chrome のアドレスバーに chrome://extensions/ をコピペ）
② 開いたページの右上「デベロッパーモード」がONになっていることを確認（OFFなら ON にする）
③ 「TimeCard Pro」のカードを探す
④ カード内の「読み込み元: /Users/...」というファイルパスを探す
⑤ そのパスをクリックすると、Finderで拡張機能フォルダが開きます🌷
　（Win の場合はエクスプローラーで開く）

《manifest.json を開く》
⑥ Finder のフォルダ内に並んでいるファイルから manifest.json を探す
⑦ manifest.json を右クリック→「このアプリケーションで開く」
⑧ 「テキストエディット」を選んでクリック（Macの場合）
　（VSCodeを持っていればVSCodeでもOK。メモ帳でも開けます）

《client_id を書き換える》
⑨ テキストエディタの上のメニューから「編集」→「検索」（または Cmd+F）
⑩ 検索ボックスに client_id と入力
⑪ "client_id": "..." という行が見つかる
⑫ ダブルクォートの中身だけを選択
⑬ STEP 4 でコピーしたあなたのIDで上書き（例：248204274658-xxxxx...apps.googleusercontent.com）
⑭ Cmd+S（Mac）/ Ctrl+S（Win）で保存

📝 完成イメージ

"oauth2": {
  "client_id": "248204274658-q1tq8i8cpau6cmfpvvdg13its8hr7861.apps.googleusercontent.com",
  "scopes": [...]
}

※ 上は例。あなたの実際のIDに置き換えてください。

💡 困ったら

・「読み込み元」が表示されない→デベロッパーモードがONか再確認（右上のトグル）
・パスをクリックしてもFinderが開かない→パス全体を選択してコピー→Finderの「移動」→「フォルダへ移動」で貼り付け
・manifest.json が見つからない→フォルダの直下にあります。サブフォルダではない
・右クリックメニューに「テキストエディット」がない→「その他」→「アプリケーション」フォルダから選択
・保存できない・エラー→「"」「,」「{」「}」を消してないか確認。緑のシールマークが付いた読み取り専用なら、ファイル情報で「読み出しと書き込み」に変更
・JSONの形式がおかしい→ChatGPTに「このmanifest.jsonの client_id を 〇〇〇 に変えて」と頼んで全文をもらうのも◎

6
拡張機能をリロードして完了🌷

📌 これは何？

manifest.json を変更したので、Chromeに「変更を反映してね」と伝える作業です。
（ChromeはmanifestをキャッシュしているのでリロードしないとIDが反映されない）

✏️ 手順

① chrome://extensions/ を開く
② TimeCard Pro のカードを探す
③ カード右下にある「↻ 更新（リロード）」アイコンをクリック
④ アイテムIDがSTEP 3 でコピーしたものと同じか念のため確認
　（違っていたら STEP 4 のクライアントIDの「アイテムID」欄を更新する必要あり）
⑤ TimeCardの設定画面で「カレンダー連携 ON」などのトグルをONにする
⑥ 一度「終了」ボタンや「📅空き時間」ボタンを押すと、Googleの認可ダイアログが出る
⑦ あなたのGoogleアカウントを選択 →「許可」を押せば完了！🌸

💡 困ったら

・「このアプリは確認されていません」と警告→「詳細」→「TimeCard（安全ではないページ）に移動」をクリック
　（自分で作った拡張なので問題なし）
・認可ダイアログが出ない→拡張をリロードした？manifestの保存ができてる？
・カレンダーに予定が投入されない→設定画面の「カレンダー連携」がONになっているか確認
・もう一度ダイアログを出したい→Googleアカウント設定の「セキュリティ」→「サードパーティのアプリ」からTimeCardの許可を取り消し、再認可

🎯 全体まとめ｜何をしているか
① Googleに「TimeCardという拡張機能を使うよ」と申請する場所＝プロジェクトを作る（STEP 1）
② 使うサービス（Sheets / Calendar）の窓口を開ける（STEP 2）
③ あなたのTimeCardを識別する身分証＝アイテムIDを取得（STEP 3）
④ 身分証を見せて鍵＝クライアントIDを発行してもらう（STEP 4）
⑤ その鍵をTimeCardに渡す（STEP 5）
⑥ 動作確認＆完了（STEP 6）

セットアップがまだの方は、ページ上部の「🚀 Google連携セットアップ手順」をご覧ください（STEP 2-2 で Calendar API を有効化）。
